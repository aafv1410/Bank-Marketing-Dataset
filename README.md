## 🛠️ Stack Tecnológico y Decisiones

Este proyecto utiliza un stack moderno centrado en la **portabilidad, automatización y escalabilidad**. A continuación, se detallan las herramientas elegidas y la razón de su uso:

### 🐍 Lenguaje y Base de Datos
* **Python:** Elegido por su versatilidad, velocidad de desarrollo y el robusto ecosistema de librerías para el manejo de datos y APIs.
* **MongoDB:** Al ser una base de datos NoSQL basada en documentos, nos permite iterar rápidamente sin las restricciones de un esquema rígido, ideal para el crecimiento ágil del proyecto.

### 📦 Infraestructura y Despliegue
* **Docker:** Implementado para garantizar que la aplicación se ejecute exactamente igual en cualquier entorno, eliminando el clásico problema de "en mi máquina funciona".
* **Render:** Elegido como plataforma de hosting por su excelente integración con Docker y su facilidad para gestionar certificados SSL y despliegues automáticos.

### ⚙️ Automatización y Control
* **Git & GitHub:** Utilizados para el control de versiones y el trabajo colaborativo mediante *Pull Requests* y revisiones de código.
* **GitHub Actions (CI/CD):** Configurado para ejecutar pruebas automáticas y desplegar la aplicación en Render cada vez que se hace un `push` a la rama principal.

### 💻 Entorno de Desarrollo
* **VS Code:** El editor estándar recomendado para este proyecto gracias a su integración nativa con Docker, Git y sus potentes extensiones para Python.
